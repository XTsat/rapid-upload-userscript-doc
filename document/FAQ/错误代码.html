# 认证失败(#-6)

  1. (秒传链接提取Ultra) 尝试先打开 百度网盘主页在新窗口打开(若未登录请先登录) 再转存
  2. (秒传链接提取Ultra) bdstoken可能已失效, 请前往设置页刷新bdstoken: 图例1在新窗口打开 图例2在新窗口打开
  3. 若使用的浏览器为safari, 请尝试改用 Mac版Chrome浏览器 (参考 安装教程)
  4. 若使用的浏览器为firefox(火狐), 且使用的油猴插件为Tampermonkey, 请尝试更换为 Violentmonkey在新窗口打开(暴力猴)再安装脚本
  5. 尝试 清空浏览器数据/重置浏览器, 若无法解决请前往 反馈页在新窗口打开 反馈: 1.浏览器版本 2.油猴插件版本 3.出现问题时按F12->打开控制台(console)->截图

# 请求失败(#514)

  1. 若生成时弹出跨域提示, 请选择允许: 图例在新窗口打开
  2. 若生成时使用了网络代理工具, 请关闭代理工具再尝试生成
  3. 尝试更新油猴插件到最新版本: 微软商店在新窗口打开 火狐商店在新窗口打开 谷歌商店在新窗口打开(可能无法访问)

# 接口限制访问(#403)

一般是账号问题, 可能是生成秒传过于频繁等各种原因导致的, 一般等待 24小时 即可解除, 也可尝试 更换账号 或 使用本地生成工具 蓝奏云在新窗口打开 密码:2233

  • 若在分享页生成秒传时弹出此错误, 请先检查 度盘账号是否登录

  • 开启 "极速生成"功能, 可有效避免此问题

# 服务器错误(#50X)

一般是百度服务器暂时问题, 可使用本地生成工具 蓝奏云在新窗口打开 密码:2233 或 1-24小时 之后再试, 若依旧出现请前往 反馈页在新窗口打开 反馈

# md5 获取失败(#996)

  1. 更新脚本到最新版本, 再尝试生成, 更新地址: 脚本页在新窗口打开
  2. IDM 拦截下载导致 "md5 获取失败", 解决方法: 图例 1在新窗口打开 , 图例 2在新窗口打开
  3. 迅雷 拦截下载导致 "md5 获取失败", 解决方法: 迅雷设置 -> 图例在新窗口打开 , 被其他下载工具拦截时同理
  4. 等待一段时间再重试, 若依旧失败, 可通过 本地生成工具 或 网页版生成工具 生成秒传, 并手动转存测试秒传有效性
    • 本地生成工具: 蓝奏云在新窗口打开 密码:2233
    • 网页版生成工具: 载点1在新窗口打开 载点2在新窗口打开 载点3在新窗口打开
  5. 若已排除1 2 3, 可尝试开启 "极速生成"功能, 可有效避免此问题

# 秒传未生效(#404)

# 秒传转存者

  1. 更新脚本到最新版本, 再尝试转存, 更新地址: 脚本页在新窗口打开
  2. 秒传可能延迟生效, 可等待一段时间(1-24 小时左右)再重试
  3. 将下方 秒传分享者 部分反馈给分享者(此未生效秒传链接的发布者)

# 秒传分享者

  1. 需要上传过秒传对应的文件到度盘且任意网盘账号内保留一份文件, 才能使秒传生效

  2. 秒传可能延迟生效, 可等待一段时间(1-24 小时左右)再重试, 若不想等待或等待后依旧无效, 可尝试重传(3.)

  3. 使用 度盘网页端 / 度盘客户端 重传文件, 再手动转存测试秒传有效性

  4. (重要) 使用脚本 2.7.3 版本生成秒传, 大概率 生成错误秒传, 导致转存报错#404, 若出现该情况请 更新脚本 并 重新生成

  5. (重要) 使用脚本 低于2.0.11 版本生成秒传, 小概率 生成错误秒传, 导致转存报错#404, 若出现该情况请 更新脚本 并 重新生成

  6. (重要) 22.10.24开始, 由于百度接口改动, 使用 BaiduPCS-Go在新窗口打开 等第三方工具上传的文件的秒传永久不生效, 出现此问题请改用官方的 度盘网页端 / 度盘客户端 上传文件

  7. (重要) 标准码格式的秒传链接可以简化, 例如:

    D5AABEFC3290F7A3C09912228B136D0C#821A9F0D27FCD19C80474D2140ED2D85#6467659#test.exe

    可以简化为 (删除第二段md5, 可以正常转存):

    D5AABEFC3290F7A3C09912228B136D0C#6467659#test.exe

    但不能简化为 (删除第一段md5, 会导致上述报错, 无法转存):

    821A9F0D27FCD19C80474D2140ED2D85#6467659#test.exe

# 转存失败-v2接口(#114)

  1. v2接口有小概率会返回该错误, 可尝试多转存几次
  2. 已知秒传数据中md5正确但size错误时也会出现该错误(即md5有效但与size不匹配)
  3. 若转存的所有文件均出现该错误, 请前往 反馈页在新窗口打开 反馈: 1.浏览器版本 2.油猴插件版本 3.出现问题时按F12->打开控制台(console)->截图

# 转存失败-v1接口(#2)

更新脚本到最新版本, 再尝试转存, 更新地址: 脚本页在新窗口打开

# 请求错误(#400)

  1. 可能为浏览器/插件问题, 可尝试参考 安装教程 安装脚本并使用
  2. 已知使用 扩容盘 账号 选中 文件夹 生成秒传时, 会出现此报错, 普通账号则不会 (见github issue在新窗口打开)
  3. 若无法解决, 请前往 反馈页在新窗口打开 反馈: 1.浏览器版本 2.油猴插件版本 3.出现问题时按F12->打开控制台(console)->截图

# 服务器错误(#31039)

  1. 更新脚本到最新版本, 再尝试转存, 更新地址: 脚本页在新窗口打开

  2. 请前往 反馈页在新窗口打开 反馈: 报错的秒传链接

  3. 尝试修改输入的秒传链接中的文件名后缀部分再转存, 例如

    D5AABEFC3290F7A3C09912228B136D0C#821A9F0D27FCD19C80474D2140ED2D85#6467659#test.rar

    修改为任意的其他后缀:

    D5AABEFC3290F7A3C09912228B136D0C#821A9F0D27FCD19C80474D2140ED2D85#6467659#test.7z

  4. 尝试使用国内ip访问度盘
